Terminal Pokémon

Ayman Habib, Period 9

  • Project Link

    • Download the game Terminal Pokémon here!

  • Project Description

    • Terminal Pokémon is a terminal, text-based version of the classic Pokémon game. You can play out Pokémon battles between yourself and a computer by typing in which Pokémon you will use and what moves you will play until one side loses all its HP.

  • Instructions

    • Use the project link above to download the Python file (You must have a program that runs Python programs). When you run the program, it will say a wild, randomly generated Pokémon has appeared, and you will then be prompted to select a Pokémon, and then pick your first move against the enemy Pokémon. The program will then return the enemy Pokémon's current health, the move the enemy Pokémon plays, and how much health the player's Pokémon has remaining. This will continue until either of the Pokémon lose all their health. Depending on which Pokémon wins, the program will have use turtles to display an image of the victorious Pokémon.

  • How it Works

    • The program uses a list of the Pokémon and their movesets coupled with a list of the total available moves and their information (the element of the move, how much damage it does, and its accuracy), both saved in a .csv file. These .csv files are then made into JSON dictionaries. The first JSON dictionary holds the Pokémon's name and their moves (denoted as move_1, move_2, etc). The second JSON dictionary holds the names of the moves and their attributes (type, how much damage it does, and accuracy).Then, the program randomly selects a Pokémon from the dictionary, which is the enemy Pokémon. The player is then prompted to select a Pokémon to play as, and then prompted to play a move. The program then returns the remaining HP of the enemy Pokémon, the move it plays, and the remaining health of the player's Pokémon. The enemy Pokémon's move is also randomly selected from a dictionary. Whether the move hits is determined by selecting a number between 0 and 100, and then checking whether this number is in the range of 0 to the accuracy. If the number is in the range, the move will hit. Otherwise, the move will miss. The player will then select another move to play, and this will repeat until one of the Pokémon lose all their HP. Afterwards, the program will have turtles display an image of the winning Pokémon.

  • Files

    • final.html: This html file
    • pokepage.html: The page containing the list of playable Pokémon, their HP, and the available moves. The moves are coupled with how much damage they do, their accuracy, and their type.
    • typepage.html: The page containing the information about how types work in this program.
    • pokemon.py: The main Python program that hosts the Terminal Pokémon game.
    • pokemon_names.csv: A csv file that contains the names of the Pokémon that can be played, their HP, and each Pokémon's moveset.
    • pokemon_moves.csv: A csv file that contains the various moves, what element type they are, how much damage they do, and the accuracy of the move.
    • gif_images/*: The .gif images for the turtles to display.
    • images/*: The images for the website.
    • pokemonfull.zip: This zip file contains all the required files for the Terminal Pokémon game. Make sure you keep all the files in the same folder or else the game will not work.

  • Bugs and Errors to be Aware of

    • You must use the same capitalization for the Pokémon names and moves as it is displayed on the website/provided to you in the program.
    • The turtle window will not close after the first round, and will not pop up to the front of the screen after the first round, but will still display the winning Pokémon's image.
